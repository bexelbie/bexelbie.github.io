# Demo 1: Provisioning a Server

Boot a Flatcar VM from a declarative config. Show that containers run
out of the box and the OS is immutable.

---

## Files in this directory

| File | What it is |
|------|------------|
| `demo1.bu` | Butane YAML — the human-readable config that declares how this machine should be set up: SSH key, hostname, static IP, a Docker container serving a website on port 8000, and update-engine masked. Uses `contents: local:` and `contents_local:` to pull in external files — not everything has to be inlined. |
| `demo1.ign` | Ignition JSON — the machine-readable version of the same config. Generated by `butane --strict --files-dir . demo1.bu`. Flatcar reads this at first boot to provision the system. |
| `resources/index.html` | The custom web page. Referenced by the Butane config via `contents: local: resources/index.html` — Butane embeds it into the Ignition JSON at transpile time. |
| `resources/my-website.service` | The systemd unit that runs the Docker container. Referenced via `contents_local: resources/my-website.service`. Keeping it as a separate file makes it easy to edit and test independently. |
| `start.sh` | Boots the VM in the background from the pre-staged work image. |
| `stop.sh` | Cleanly shuts down the VM via SSH (falls back to kill). |
| `go.sh` | SSH into the running VM. |
| `status.sh` | Shows whether the VM is running and guest details. |
| `assets/` | Base image, compressed image, and pre-staged work image (with Docker's nginx:alpine already cached so no internet is needed at demo time). |

## Troubleshooting

**SSH says "Too many authentication failures"**
— Your SSH agent has many keys loaded. The scripts use
`-o IdentitiesOnly=yes` to avoid this. If running SSH manually, add
that flag or use `go.sh`.

**Container not running on second boot**
— Check `systemctl status my-website.service` and `docker logs my-website`.
If the Docker image got lost, you need to re-stage from a fresh first
boot with internet access.

**VM won't boot (no work image)**
— `start.sh` creates the work image from the base image automatically.
If the base image is missing, re-download stable from
`https://stable.release.flatcar-linux.net/amd64-usr/current/flatcar_production_qemu_image.img.bz2`.
